You can use COM Components in .NET Apps, but have you ever wished for the other way, Use of .NET Classes you have written in VB6 a apps. Yes, this is possible, just read this tutorial.

Step 1: Create You Own .NET Class Library Project

Step 2: Open The project properties > Configuration Properties > Build and Check Register For COM Interop

Step 3: Open the Visual Studio 2003 Command Prompt (either from start menu or by running vsvars32.bat found in the Common7\Tools folder of VS folder.)

Step 4: Register the .NET DLL by running REGASM "PATH TO .NET DLL". This will register the .NET DLL as COM DLL ready to be called from COM languages like VB6

Step 5: The DLL will appear in the VB6 reference dialog, add in to your vb6 project and use the DLL as usual.

Limitations:

  1. Intellisence do not work
  2. The VB6 app needs to be complied ever time you make changes to the .NET DLL
  3. .NET Control cannot be used, only Class Libraries

Why Use This Method:

  1. Its Cost & Time Effective
    Why Convert/Rewrite the whole app to .NET if you only want to use the NEW features of it e.g. webservices etc.
  2. Not Every VB6 app can be ported to .NET
    Because of major changes in the language/platform not every app can be updated. So the only way is to break the app into smaller parts and move to .NET
  3. Support/Maintain You Legacy Apps
    Using this method you can easily improve & support/maintain your legacy COM app by purchasing .NET components without complete rewrite/conversion
